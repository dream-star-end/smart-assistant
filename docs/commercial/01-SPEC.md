# 01 业务需求规格(SPEC)

## 1. 产品定位

OpenClaude 商业版 = **对话式 AI 助手 + 可选的 Agent 工作空间**,面向付费个人用户。

- **对话模式**(免费 + 积分付费):无状态聊天,按 token 扣积分
- **Agent 模式**(月费开通 + 积分付费):每用户独立、持久化的隔离工作空间,支持工具/命令/文件操作;仍按 token 扣积分

底层账号池使用 Claude 订阅账号(OAuth token),用户无感,超管可管理。

## 2. 用户角色

| 角色 | 能力 |
|------|------|
| **访客** | 浏览首页、注册、登录 |
| **普通用户 (user)** | 登录、充值积分、对话模式聊天、开通 Agent、使用 Agent 工作空间、查看自己的账单流水、修改资料、注销 |
| **超级管理员 (admin)** | 全部用户能力 + 用户管理 + 账号池管理 + 倍率配置 + 流水审计 + Agent 实例监控 + 系统状态 |

无中间角色(无财务/运维/客服拆分)。

## 3. 功能需求(MVP 范围)

### F-1 用户系统
- F-1.1 用户注册:邮箱 + 密码 + Turnstile 验证码
- F-1.2 邮箱验证:注册后发送验证链接(24h 有效)
- F-1.3 登录:邮箱 + 密码 → JWT(access 15min + refresh 30d)
- F-1.4 登出:吊销 refresh token
- F-1.5 密码重置:邮箱发验证码,验证后改密
- F-1.6 修改资料:昵称、头像(可选)、修改密码
- F-1.7 账户注销:软删除 + 30 天冷却后硬删除

### F-2 积分计费
- F-2.1 实时扣费:LLM 请求完成后,按 `usage_tokens × 模型单价 × 倍率` 计算积分 cost,写 ledger
- F-2.2 价格快照:扣费时冻结当时的 `model_pricing` 快照,存入 `usage_records.price_snapshot`
- F-2.3 4 维计费:分别扣 input / output / cache_read / cache_write 的 token
- F-2.4 余额预检:请求前按 `max_tokens` 估算上限,余额不足直接拒绝
- F-2.5 账单查询:用户可查自己所有流水;超管可查全部
- F-2.6 流水不可篡改:`credit_ledger` 表禁 UPDATE/DELETE(DB 层约束)

### F-3 充值与支付
- F-3.1 虎皮椒微信扫码支付:生成订单 → 返回二维码 → 用户扫 → 回调加积分
- F-3.2 订单幂等:同 `trade_order_id` 只处理一次
- F-3.3 超时关闭:15 分钟未支付自动 cancel
- F-3.4 每日对账:拉虎皮椒订单列表 vs 本地,差异告警
- F-3.5 退款:超管手工操作,前台不提供自助退款
- F-3.6 充值套餐:MVP 固定 4 档(¥10 / ¥50 / ¥200 / ¥1000),对应积分 1:1 或带赠送比例

### F-4 对话模式(Chat Mode)
- F-4.1 Web UI 聊天界面:消息列表、输入框、流式输出
- F-4.2 模型选择:MVP 只放开 Claude 主流模型(sonnet/opus)
- F-4.3 会话历史:存储用户所有对话,可检索、删除
- F-4.4 会话并发:同一用户最多 3 个并发 chat 请求
- F-4.5 上下文长度:Gateway 控制历史长度,超限自动裁剪

### F-5 Agent 模式(MVP 基础档)
- F-5.1 开通:用户 ¥29/月 订阅,扣积分自动续费(或手动)
- F-5.2 资源配额(基础档唯一):
  - CPU: 0.2 core
  - Memory: 384 MB
  - Disk: 5 GB(workspace + home)
  - 网络:白名单出口 + 透明代理
- F-5.3 隔离:每用户独立 Docker container(`agent-u{uid}`)
- F-5.4 持久化:用户 workspace(`/workspace`)和 home(`/root`)独立 volume,容器重启不丢
- F-5.5 常驻:开通期内 container 持续 running(空闲不销毁)
- F-5.6 到期处理:订阅过期 → container stop,volume 保留 30 天 → 30 天未续 → volume 清理
- F-5.7 Agent 使用:Web UI 提供 Agent 会话入口,进入后 WebSocket 连接到用户专属 container
- F-5.8 Agent 计费:Agent 产生的 LLM 请求照常按 F-2 扣积分(月费只买环境,不含 LLM)
- F-5.9 Agent 审计:所有工具调用(bash/file/web/...)落日志,超管可查
- F-5.10 唯一实例:每用户最多 1 个 Agent container

### F-6 账号池(底座,用户不感知)
- F-6.1 账号 CRUD:超管添加/编辑/删除 Claude OAuth 账号
- F-6.2 Token 加密存储:AES-256-GCM,密钥从环境变量注入
- F-6.3 健康度追踪:每账号记录 success/fail、剩余配额、cooldown
- F-6.4 调度策略:
  - Agent 会话 sticky(同一会话固定账号,保 prompt cache)
  - Chat 会话加权轮询(按剩余配额加权)
- F-6.5 熔断:连续 3 次 429/5xx → cooldown 10min,health -= 20
- F-6.6 半开恢复:cooldown 到期试探,成功 health += 10
- F-6.7 Token 过期自动 refresh
- F-6.8 全部账号不可用 → 返回 503 + 告警

### F-7 超管后台
- F-7.1 用户管理:列表、搜索、查看详情、封禁/解封、调整积分(带审计)
- F-7.2 账号池管理:添加/编辑/删除账号、查看健康度、手工 cooldown
- F-7.3 定价管理:模型单价 + 倍率调整(立即生效但不影响历史账单)
- F-7.4 流水审计:所有积分变动可查,按用户/时间过滤
- F-7.5 Agent 实例监控:当前运行的 container 列表、资源占用、手工重启/停止
- F-7.6 系统状态:账号池健康、DB 状态、Redis、近期错误
- F-7.7 超管操作审计:所有超管动作写 `admin_audit_log`,append-only

### F-8 安全与反滥用(贯穿所有)
- 详见 [05-SECURITY.md](./05-SECURITY.md)

## 4. 非功能需求

### N-1 性能
- API p95 延迟 < 300ms(不含 LLM 等待)
- Gateway 单实例支撑 500 并发连接
- DB 连接池上限 50
- Agent container 冷启动 < 5s(v2 再优化)

### N-2 可用性
- MVP 单机部署,可接受 < 99% 可用性
- 服务重启不丢失会话(会话持久化)
- 账号池单点故障不影响其他用户

### N-3 可观测性
- 结构化日志(JSON),按请求 trace_id 串联
- 关键指标埋点:QPS、扣费成功率、账号池健康、容器状态
- 超管后台实时展示

### N-4 合规
- 用户数据可导出、可删除(合规要求)
- 支付符合虎皮椒规范
- 账号池运营风险自担(见 05-SECURITY)

## 6. 验收标准(何时可上线)

- ✅ 一个真实用户能在 `https://claudeai.chat/` 完成:注册 → 邮箱验证 → 登录 → 充值 → 对话
- ✅ 一个真实用户能开通 Agent 基础档 → 进入 Agent 会话 → 执行 bash/文件操作 → 关闭浏览器 24h 后回来文件还在
- ✅ 超管能在后台:看到该用户、手动加减积分、查他的流水、看到他的 container 状态
- ✅ 账号池中一个账号被 429 封禁 → 自动熔断 → 其他账号接管请求 → 用户无感
- ✅ 所有测试通过(`npm run check`)
- ✅ 无严重安全漏洞(见 05-SECURITY 清单逐项打勾)
